El **Gestor de Visitas del Túnel Subfluvial** es la plataforma operativa que permite al personal registrar y gestionar visitas grupales de manera eficiente. Desde el agendamiento de una nueva visita hasta la generación de reportes gerenciales en PDF, el sistema centraliza toda la operación en una interfaz clara y segura.

## Primeros pasos

<Steps>
  <Step title="Instalar el sistema">
    Clona el repositorio, configura las variables de entorno y levanta el backend con `npm run dev` y el frontend con `npm run dev`.
    
    Consulta la [guía de instalación](/instalacion) para instrucciones detalladas.
  </Step>
  <Step title="Iniciar sesión">
    Accede con tus credenciales de usuario. El sistema soporta dos roles: **Admin** (acceso completo) y **Guía** (registro y consulta de visitas).
    
    Consulta la [guía de autenticación](/guias/autenticacion) para más detalles.
  </Step>
  <Step title="Registrar una visita">
    Desde el Dashboard o el botón "Nueva Visita", completa el formulario con la fecha, hora, tipo, gestor y cantidad de personas. El sistema valida el aforo automáticamente.
    
    Consulta la [guía de registro de visitas](/guias/registro-visitas).
  </Step>
  <Step title="Explorar la API">
    Si estás integrando el sistema con otra herramienta, consulta la [Referencia API](/api/descripcion-general) para conocer todos los endpoints disponibles.
  </Step>
</Steps>

## Roles y permisos

El sistema define dos roles principales:

| Rol | Acceso |
|-----|--------|
| **Guía** | Dashboard, registro y edición de visitas, consulta de historial, gestión de gestores, exportación de comprobantes |
| **Admin** | Todo lo anterior + gestión de usuarios, configuración del sistema, días inhábiles, estadísticas gerenciales, auditoría y exportación de informes |

<Note>
  La sesión se cierra automáticamente tras un período de inactividad configurable. Por defecto son **30 minutos**. Los administradores pueden ajustar este valor desde **Configuraciones del Sistema**.
</Note>
